Control de Versiones con Git

Git es un sistema de control de versiones distribuido, ampliamente utilizado en el desarrollo de software moderno. Fue creado para gestionar proyectos de forma rápida, eficiente y colaborativa.

¿Sabías? Git fue creado por Linus Torvalds en 2005 para el desarrollo del kernel de Linux.

Comandos más populares de Git

  • git init — Inicializa un nuevo repositorio Git en el directorio actual. Este comando crea un repositorio vacío.
  • git clone [url] — Clona un repositorio remoto en tu máquina local. Ejemplo: git clone https://github.com/usuario/repositorio.git.
  • git status — Muestra el estado del repositorio: archivos modificados, nuevos archivos no rastreados, cambios preparados para el commit, etc.
  • git add [archivo] — Agrega un archivo o cambios específicos al área de preparación (staging area) para el commit.
  • git commit -m "[mensaje]" — Crea un commit con los archivos añadidos al área de preparación, agregando un mensaje descriptivo.
  • git push — Sube los cambios locales al repositorio remoto. Generalmente se usa después de un commit.
  • git pull — Obtiene y combina los cambios del repositorio remoto al local. Equivalente a git fetch seguido de git merge.
  • git checkout [rama] — Cambia a una rama específica. Si no existe, se puede crear con git checkout -b [rama].
Ramas

Las ramas (branches) permiten trabajar en nuevas funcionalidades sin afectar el código principal. Son una de las características más poderosas de Git.

Merge

El merge combina los cambios de una rama en otra. Git intenta hacer esto automáticamente, pero a veces pueden surgir conflictos que hay que resolver manualmente.

Repositorios remotos

Plataformas como GitHub, GitLab o Bitbucket permiten alojar repositorios en la nube para colaborar con otros desarrolladores de todo el mundo.

Conceptos clave

Un commit es una instantanea del estado de tu proyecto en un momento determinado. Cada commit tiene un identificador único (hash) y un mensaje descriptivo.

El staging area (área de preparación) es una zona intermedia donde se van acumulando los cambios que se incluirán en el próximo commit, usando git add.

Git es el sistema de control de versiones que corre localmente en tu computadora. GitHub es una plataforma web que permite alojar repositorios Git en la nube y colaborar con otros.
